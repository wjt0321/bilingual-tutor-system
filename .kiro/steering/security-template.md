# 安全性与隐私保护规范模板

## 说明

本文件是 `security.md` 的模板版本。实际的 `security.md` 文件包含具体的隐私保护规范，但为了安全考虑，该文件仅保存在本地，不会上传到公共仓库。

## 用途

`security.md` 文件用于：

1. **隐私信息脱敏**：定义如何将真实的个人信息替换为示例值
2. **上传前检查**：提供安全检查清单，防止敏感信息泄露
3. **AI助手规范**：指导AI助手在处理上传任务时的安全行为
4. **应急处理**：提供敏感信息意外泄露时的处理方法

## 创建说明

如果您需要在新环境中创建 `security.md` 文件，请：

1. 复制此模板文件
2. 重命名为 `security.md`
3. 根据您的具体需求填写详细的安全规范
4. 确保该文件已添加到 `.gitignore` 中

## 重要提醒

- `security.md` 文件应始终保持本地专用
- 不要将包含真实敏感信息示例的安全规范上传到公共仓库
- 定期检查 `.gitignore` 确保安全文件不会被意外提交

## 技术实现

该文件通过以下方式确保安全：

```gitignore
# 在 .gitignore 中添加
.kiro/steering/security.md
```

这样可以确保：
- 文件在本地存在并生效
- Kiro可以读取并应用安全规范
- 文件不会被上传到GitHub等公共平台
- 保护用户隐私信息不被泄露
